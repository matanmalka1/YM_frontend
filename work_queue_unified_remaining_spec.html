# Work Queue + Manual Tasks — מה נותר לבצע ואיחוד למסך אחד ## 1. החלטה מוצרית חדשה לא נשארים עם שני מסכים נפרדים: ```txt מסך עבודה לטיפול מסך משימות ידניות ``` עוברים למסך אחד: ```txt עבודה לטיפול ``` המסך הזה יהיה המקום היחיד שבו משתמש רואה, יוצר, עורך, מתחיל, משלים, מבטל ומנהל משימות ידניות — וגם מטפל בפריטי מערכת כמו דוחות, חיובים, מקדמות וקלסרים. אבל חשוב: ```txt לא הופכים כל פריט מערכת ל־Task. ``` המסך המאוחד יציג שני סוגי דברים: ```txt 1. עבודות מערכת 2. משימות ידניות ``` ההבדל חייב להיות ברור ב־UI וב־API. --- ## 2. ההגדרה הנכונה ### עבודת מערכת עבודת מערכת היא פריט שמגיע מדומיין קיים: ```txt vat_work_item annual_report advance_payment charge binder ``` היא נוצרת מהמצב האמיתי של הדומיין. דוגמאות: ```txt דוח מע״מ שלא הוגש דוח שנתי פתוח מקדמה שלא שולמה חיוב שהונפק ולא שולם קלסר שממתין לאיסוף ``` הפעולה העיקרית עליה: ```txt פתח מקור צור משימה קשורה ``` לא עורכים אותה כמו Task. --- ### משימה ידנית משימה ידנית היא שורת `Task`. היא יכולה להיות: ```txt משימה כללית משימה שמקושרת לדוח משימה שמקושרת לחיוב משימה שמקושרת ללקוח משימה פנימית לעובד ``` הפעולות עליה: ```txt צור ערוך התחל השלם בטל מחק / העבר לארכיון ``` --- ## 3. מצב נוכחי אחרי cleanup בוצע: ```txt Work Queue הוא projection מחושב. source types נוקו. legacy aliases הוסרו. payload הוחלף ב־metadata. שדות כפולים הוסרו. actions לא בטוחות הוסרו. נשארו link actions בלבד למקורות מערכת. ל־task נשארו start / complete / cancel. נוסף grouping בסיסי. ``` Source types סופיים: ```txt vat_work_item annual_report advance_payment charge binder task ``` מה עדיין חסר: ```txt יצירת משימה מתוך source עריכת משימה מתוך המסך מחיקת משימה פתיחת משימה / drawer ניהול מלא של משימות ידניות במסך המאוחד הסרת מסך משימות ידניות הישן מהניווט / איחוד מלא UI שמבדיל ברור בין system work לבין manual tasks ``` --- # 4. מה לא בוצע עדיין ## 4.1 Create linked task כרגע אין פעולה: ```txt צור משימה ``` על שורת מערכת. דוגמה רצויה: ```txt דוח שנתי 2023 של לקוח X [פתח דוח] [צור משימה] ``` לחיצה על `צור משימה` פותחת modal. השמירה יוצרת Task עם: ```txt source_domain = item.source_type source_id = item.source_id ``` אחרי השמירה: ```txt השורה נשארת source row המשימה מופיעה תחת linked_tasks אין כפילות ``` --- ## 4.2 Edit task כרגע אין עריכת משימה מתוך Work Queue. צריך לאפשר עריכה עבור: ```txt task standalone linked task בתוך source row ``` שדות עריכה: ```txt כותרת תיאור תאריך יעד עדיפות שיוך למשתמש / תפקיד ``` לא לערוך: ```txt source_domain source_id ``` אלא אם יש החלטה מפורשת לאפשר שינוי קישור. ברירת מחדל: לא משנים קישור source בעריכה רגילה. --- ## 4.3 Delete task כרגע אין מחיקת Task. צריך להחליט: ```txt soft delete בלבד ``` לא מחיקה פיזית. נדרש endpoint: ```http DELETE /api/v1/tasks/{task_id} ``` או endpoint קיים אם יש. מחיקה עושה: ```txt deleted_at = now ``` אם קיימים שדות audit: ```txt deleted_by ``` אם אין — לא להוסיף סתם אלא אם יש pattern קיים במערכת. אחרי מחיקה: ```txt task לא מופיעה במסך source ממשיך להופיע אם הוא עדיין דורש טיפול ``` --- ## 4.4 Open task / Task drawer אם מאחדים מסכים, צריך דרך לפתוח משימה. לא חייב route: ```txt /tasks/:id ``` אפשר drawer/modal בתוך Work Queue. המלצה: ```txt Task drawer צדדי ``` שימושים: ```txt צפייה בפרטי משימה עריכה הצגת source מקושר פעולות start/complete/cancel/delete ``` --- ## 4.5 מסך משימות ידניות הישן צריך להחליט מה קורה איתו. אם המטרה היא מסך אחד בלבד: ```txt להסיר מהניווט להפנות route ישן ל־/work-queue או למחוק route אם אין שימוש ``` אבל לא למחוק מיד קוד אם הוא מכיל form/use hooks שצריך להעביר. נדרש: ```txt לחלץ רכיבי Task form לשימוש חוזר להעביר יכולות למסך Work Queue אחרי זה למחוק את המסך הישן ``` --- ## 4.6 Grouping עדיין לא מספיק כרגע grouping בסיסי לפי: ```txt source_type + period/tax_year + due_date ``` צריך לשפר: ```txt כותרת קבוצה ברורה סיכום כמות כמות עם משימה קשורה כמות warnings פעולה קבוצתית אם רלוונטי פתיחה/סגירה של קבוצה ``` דוגמה: ```txt דוחות שנתיים 2023 · יעד 31/05/2024 12 לקוחות · 3 עם משימה קשורה · 1 אזהרה ``` --- ## 4.7 הפרדה ויזואלית חסרה במסך המאוחד חייבים להבדיל בין: ```txt עבודת מערכת משימה ידנית עבודת מערכת עם משימה קשורה משימה ידנית כללית משימה ידנית שמקושרת ל־source שטופל ``` אם לא, המשתמש לא יבין מה הוא מוחק/עורך. --- # 5. מבנה המסך המאוחד ## 5.1 Header שם המסך: ```txt עבודה לטיפול ``` תיאור: ```txt כל מה שדורש טיפול: דוחות, חיובים, מקדמות, קלסרים ומשימות ידניות. ``` כפתור ראשי: ```txt + משימה ידנית ``` הכפתור יוצר Task כללית, ללא source. --- ## 5.2 Summary cards כרטיסים מומלצים: ```txt באיחור דחוף חשוב משימות ידניות עם משימה קשורה אזהרות ``` לא להציג כרטיס שאין לו משמעות. אם רוב הדברים באיחור — להוסיף חלוקה לפי סוג עבודה. --- ## 5.3 Filters פילטרים: ```txt חיפוש סוג עבודה דחיפות מצב טיפול עם / בלי משימה קשורה משימות ידניות בלבד עבודות מערכת בלבד לקוח ``` מצבי טיפול: ```txt לא התחיל יש משימה פתוחה בטיפול הושלם source אבל task פתוחה בעיה / warning ``` --- ## 5.4 תצוגה ראשית ברירת מחדל: ```txt Grouped view ``` לא flat table. קבוצות לפי: ```txt source_type period / tax_year due_date ``` בתוך כל קבוצה: ```txt לקוח מס׳ לקוח סטטוס source מצב טיפול משימה קשורה פעולות ``` --- ## 5.5 שורת מערכת ללא task דוגמה: ```txt דוח שנתי 2023 | אבי ישראלי | באיחור | אין משימה ``` Actions: ```txt פתח מקור צור משימה ``` אין: ```txt ערוך מחק השלם ``` כי זו לא task. --- ## 5.6 שורת מערכת עם task קשורה דוגמה: ```txt דוח שנתי 2023 | אבי ישראלי | באיחור | משימה: להשיג מסמכים ``` Actions: ```txt פתח מקור פתח משימה ערוך משימה התחל משימה השלם משימה בטל משימה ``` אם יש כמה tasks: ```txt תפריט "משימות קשורות" ``` כל task בתפריט עם פעולות משלה. --- ## 5.7 שורת task standalone דוגמה: ```txt משימה כללית | להתקשר ללקוח לגבי מסמכים ``` Actions: ```txt פתח ערוך התחל השלם בטל מחק ``` אין `פתח מקור` אם אין source. --- # 6. התנהגות עסקית ## 6.1 יצירת משימה כללית משתמש לוחץ: ```txt + משימה ידנית ``` נפתח modal. נשמרת Task בלי: ```txt source_domain source_id ``` אחרי שמירה: ```txt מופיעה בקבוצה "משימות ידניות" ``` --- ## 6.2 יצירת משימה מקושרת משתמש לוחץ על source row: ```txt צור משימה ``` נפתח modal עם context: ```txt מקושרת אל: דוח שנתי 2023 · לקוח X ``` נשמרת Task עם: ```txt source_domain = source_type source_id = source_id ``` אחרי שמירה: ```txt source row מתעדכן linked_tasks_count עולה לא נוצרת שורת task נוספת ``` --- ## 6.3 עריכת משימה עריכה משנה רק שדות task: ```txt title description due_date priority assigned_to_user_id assigned_role ``` לא משנה source. --- ## 6.4 השלמת משימה אם משלימים task מקושרת: ```txt ה־source לא נעלם אוטומטית ``` הוא ייעלם רק אם הדומיין המקורי טופל. דוגמה: ```txt השלמת task "להתקשר ללקוח" לא מגישה דוח שנתי ``` --- ## 6.5 ביטול משימה ביטול task מקושרת: ```txt מוריד אותה מ־linked_tasks source נשאר אם עדיין דורש טיפול ``` --- ## 6.6 מחיקת משימה מחיקה: ```txt soft delete ``` לא משפיעה על source. --- ## 6.7 source טופל ויש task פתוחה אם source כבר final אבל task עדיין פתוחה: ```txt להציג task standalone עם warning ``` Warning: ```txt הפריט המקושר כבר טופל ``` Actions: ```txt פתח משימה ערוך השלם בטל מחק ``` --- ## 6.8 source נמחק ויש task פתוחה להציג task standalone עם warning: ```txt הפריט המקושר נמחק או לא נמצא ``` לא להפיל את המסך. --- # 7. Backend — מה נדרש להשלים ## 7.1 Task actions מלאות להחזיר actions עבור task: ```txt open_task edit_task start_task complete_task cancel_task delete_task ``` אבל רק אם יש backend/frontend אמיתי. אם open/edit הם drawer בפרונט, action יכול להיות: ```txt type = ui ``` או להשתמש ב־link/modal action contract. צריך להחליט חוזה. אפשרות פשוטה: ```ts type: 'link' | 'mutation' | 'modal' ``` ל־edit/open: ```txt type = modal key = edit_task ``` --- ## 7.2 Create linked task צריך יכולת backend ליצור task עם source. אפשרות A: ```txt להשתמש ב־POST /tasks הקיים אם הוא כבר תומך source_domain/source_id ``` אפשרות B: ```http POST /api/v1/tasks/from-source ``` עדיף לא להוסיף endpoint אם create רגיל מספיק. דרישות: ```txt validate source_type חוקי validate source_id קיים אם אפשר לא ליצור task אם source_type לא מוכר ``` --- ## 7.3 Delete task להוסיף endpoint delete אם חסר. ```http DELETE /api/v1/tasks/{id} ``` Soft delete. להחזיר 204 או task מעודכנת לפי pattern קיים. --- ## 7.4 WorkQueue merge לוודא שה־merge תומך: ```txt source + linked task source + multiple linked tasks task linked source not in queue source final + task open source deleted + task open task standalone ``` כבר יש חלק גדול מזה. --- ## 7.5 available_actions צריך לבנות actions לפי מצב: ### source בלי linked tasks ```txt open_source create_linked_task ``` ### source עם linked task ```txt open_source open_task edit_task start_task complete_task cancel_task ``` ### task standalone ```txt open_task edit_task start_task complete_task cancel_task delete_task ``` לא להחזיר action שאינה אפשרית לפי status. --- # 8. Frontend — מה נדרש להשלים ## 8.1 Task modal לבנות או להעביר modal קיים. צריך לתמוך: ```txt create standalone task create linked task edit task view task ``` לא לבנות שלושה modals נפרדים אם אפשר רכיב אחד פשוט. --- ## 8.2 Header action להוסיף: ```txt + משימה ידנית ``` פותח TaskModal במצב create standalone. --- ## 8.3 Row action: צור משימה על source row בלי task: ```txt צור משימה ``` פותח TaskModal במצב create linked. --- ## 8.4 Row action: ערוך משימה על task או linked task: ```txt ערוך ``` פותח TaskModal במצב edit. --- ## 8.5 Row action: מחק משימה על task בלבד: ```txt מחק ``` פותח confirm. לא להציג על source. --- ## 8.6 Linked tasks UI אם יש task אחת: ```txt להציג שם task + סטטוס + פעולות מהירות ``` אם יש כמה: ```txt להציג count תפריט נפתח עם כל task ``` --- ## 8.7 הסרת מסך tasks הישן לא ישר למחוק. סדר נכון: ```txt 1. להעביר form/hooks שימושיים ל־shared 2. להשתמש בהם ב־Work Queue 3. לוודא שכל יכולות tasks קיימות במסך החדש 4. להסיר route מהניווט 5. להפנות route ישן ל־work-queue או למחוק 6. למחוק קוד ישן שלא בשימוש ``` --- # 9. מה לא לבקש / לא לבצע עכשיו לא לבקש טסטים בפרונט. במקום זה: ```txt להריץ typecheck להריץ build לעשות בדיקה ידנית מסודרת במסך ``` לא להוסיף mutation דומייניות עכשיו: ```txt file VAT submit annual report mark advance paid charge mark paid binder return ``` אלה יחזרו רק אחרי אפיון נפרד. --- # 10. בדיקות ידניות נדרשות אחרי המימוש צריך לבדוק ידנית: ## 10.1 יצירת משימה כללית ```txt לחיצה על + משימה ידנית מילוי פרטים שמירה המשימה מופיעה במסך ``` ## 10.2 יצירת משימה מקושרת ```txt שורת דוח שנתי לחיצה צור משימה שמירה השורה מציגה משימה קשורה אין כפילות ``` ## 10.3 עריכת משימה ```txt ערוך task שנה כותרת / due date שמור המסך מתעדכן ``` ## 10.4 start/complete/cancel ```txt התחל סטטוס משתנה לבטיפול השלם המשימה נעלמת או עוברת לפי policy בטל המשימה לא מופיעה כפתוחה ``` ## 10.5 מחיקה ```txt מחק task confirm המשימה נעלמת source נשאר אם עדיין פתוח ``` ## 10.6 source final + task open ```txt source טופל task עדיין פתוחה מופיעה עם warning ``` --- # 11.
